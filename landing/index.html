Local & Private Embeddings

Core Capabilities

Built from the ground up for agentic workflows, resolving common context and semantic search pain points.

🛡️

100% Local Privacy

All source code parsing, tree-sitter chunking, and ChromaDB vector embeddings run directly on your own hardware via Ollama or LM Studio. Zero source code is sent to external APIs.

⚡

True Hybrid Search

Combines classical BM25 keyword matching (FTS5) with semantic vector search. Reranks the top results using Reciprocal Rank Fusion (RRF) for ultimate code search accuracy.

🌲

Omni-Language Tree-sitter

Understands structural code hierarchies (classes, methods, variables) across Python, TS/JS, React, Rust, Go, Java, Swift, C#, C++, and more. Fetches outline definitions instead of full files.

🧠

Zero-Prompt Memory Auto-Resume

Durable session hibernation automatically creates a .fourtindex_handoff.md and injects instructions, allowing AI agents to instantly pick up right where they left off in new sessions.

📝

Local File Summarization

Encountering massive files? FourTIndex utilizes your local LLM to summarize large code components into short, information-packed tokens, saving precious context space.

⚙️

Zero-Config Agent Skills

Auto-injects instructions (SKILL.md) to instantly teach your agent how to use the MCP tools. Fits seamlessly into Cline, Cursor, Claude Desktop, and Antigravity.

Zero-Configuration MCP Integration

Plug FourTIndex directly into your favorite agentic tools in less than 60 seconds.

Add a new stdio MCP server in Cursor settings or Cline's MCP Configuration panel:

{
  "mcpServers": {
    "fourtindex": {
      "command": "/absolute/path/to/FourTIndex/.venv/Scripts/python.exe",
      "args": ["/absolute/path/to/FourTIndex/main.py", "mcp"],
      "env": {
        "PYTHONPATH": "/absolute/path/to/FourTIndex"
      }
    }
  }
}
* Note: Replace python.exe path with "python" on macOS/Linux.

Add the server configuration block to your claude_desktop_config.json:

{
  "mcpServers": {
    "fourtindex": {
      "command": "/absolute/path/to/FourTIndex/.venv/Scripts/python.exe",
      "args": ["/absolute/path/to/FourTIndex/main.py", "mcp"],
      "env": {
        "PYTHONPATH": "/absolute/path/to/FourTIndex"
      }
    }
  }
}
* Windows config is located at: %APPDATA%\Claude\claude_desktop_config.json

Run commands directly using the CLI tool in your terminal:

# 1. Setup Ollama embedding model
fourtindex setup-ollama

# 2. Index your current codebase directory
fourtindex index .

# 3. Search semantically
fourtindex search "how is batch split logic implemented?"

Get Started in 3 Simple Steps

Install, connect a local LLM/embedding provider, and index your project repository.

1

Clone & Install

Clone the repository and install the Python package in an active virtual environment:

git clone https://github.com/Chunn241529/FourTIndex.git
cd FourTIndex
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -e .
2

Configure Local Model

Configure local vector embedder using either Ollama or LM Studio:

# For Ollama (uses nomic-embed-text)
fourtindex setup-ollama

# OR For LM Studio
fourtindex setup-lmstudio
3

Index Workspace

Navigate to any developer workspace and kick off the indexing process:

cd /path/to/my-project
fourtindex index .
